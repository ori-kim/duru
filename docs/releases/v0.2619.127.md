## Highlights

- Added configured header injection for API-style targets and multipart form upload support.
- Reworked the extension lifecycle API around `registerHook(event, handler)` with `cli-start`, `target-start`, `target-end`, and `cli-end`.
- Moved command history into the `history` extension, backed by local JSONL partitions under the clip config directory.
- Improved `clip history` for terminal use: concise list rows, full detail by id, `pick`/`fzf` support, and zsh completion for recent records.
- Made CLI errors friendlier by default while keeping stack traces available through debug mode.

## Breaking Changes

- Legacy lifecycle hook names were removed. Extensions should call `registerHook()` with the new event names.
- History is no longer a built-in CLI feature; it is provided by the bundled `history` extension.

## Developer Notes

- Target extensions can now contribute configured headers consistently across API, GraphQL, gRPC, MCP, script, and CLI protocols.
- History recording runs from the `cli-end` hook, redacts sensitive-looking arguments, creates the history directory on first write, and skips `clip history` commands.
- History table output summarizes large `--args`/JSON payloads; `clip history show <id>` displays the full stored command.
- Extension docs were updated in English and Korean with the new hook event table and lifecycle diagram.
