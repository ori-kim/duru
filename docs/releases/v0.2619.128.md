## Highlights

- Formalized the extension command API around `api.commands.register()` and `api.commands.override()`.
- Added `api.options.registerGlobal()` so extensions can contribute global flags consistently.
- Added `clip update` as a protected early built-in command that can check or install GitHub release binaries before user extensions load.
- Kept top-level `clip --help` and `clip --version` on the early path so broken local extensions do not block diagnostics.
- Migrated bundled extensions (`history`, `context-mode`, `recap`, `skills`, and `sanitizer`) to the latest command and hook APIs.
- Updated extension scaffolding, installer metadata, completion/list output, and docs to use `contributes.commands`.

## Breaking Changes

- `api.registerInternalCommand()` and `api.overrideInternalCommand()` were removed.
- Manifest `contributes.internalCommands` was removed. Use `contributes.commands` instead.
- Built-in command replacement now requires `api.commands.override(name, spec)`. The previous `allowVerbOverride()` compatibility path was removed.
- Command lifecycle records now use command-oriented naming; top-level commands are exposed as `kind: "command"` in command hook summaries.
- Extensions using old lifecycle names such as `cli-start`, `cli-end`, `target-start`, or `target-end` must move to `command-start`, `command-end`, `subcommand-start`, and `subcommand-end`.

## Migration Notes

```ts
api.commands.register({
  name: "hello",
  async run({ args, options, globalOptions }) {
    console.log({ args, options, globalOptions });
  },
});
```

```yaml
contributes:
  commands: [hello]
```
